# 游戏 UI 生图最佳实践

## 1. 先定义 UI 类型

不同 UI 类型的布局重点不同：

- 活动弹窗：主面板、标题、奖励区、行动按钮、关闭按钮。
- 商城礼包页：商品卡、价格、折扣、购买按钮、货币条。
- 任务面板：任务列表、进度、奖励、领取按钮。
- 战令界面：等级进度、奖励轨道、付费/免费区分。
- 抽卡界面：卡池 banner、抽取按钮、货币、概率/次数提示。
- HUD：血条、技能按钮、小地图、资源条，要求边缘安全区域清晰。
- 结算页：胜负标题、奖励、统计、继续按钮。

不要让一个画面同时承担太多 UI 类型。

## 2. 画布和安全边距

推荐：

- 竖屏：`720x1280` 或 `1080x1920`
- 横屏：`1280x720` 或 `1920x1080`

准则：

- 主体 UI 不要贴边，保留安全边距。
- 关闭按钮、货币条等边角元素不要被裁切。
- hero 级标题可以突出，但不能挡住核心内容。
- 如果用于后续 Layout IR，优先保持正交 UI，不要强透视。

## 3. 元素要可拆

为了后续拆 sprite，prompt 中要明确：

- UI 元素边界清晰
- 元素之间有适当间距
- 面板、标题牌、按钮、卡片、icon、进度条视觉上可分离
- 不要把文字、icon、背景花纹糊成一体
- 不要让背景装饰穿过核心 UI 边缘

错误倾向：

- 光效压满所有层
- 装饰和按钮融在一起
- 文字贴在复杂纹理上
- icon 被卡片边框严重遮挡
- 主面板和背景颜色太接近

## 4. 文字策略

文字是 AI UI 生图最容易出问题的部分。

准则：

- prompt 中列出精确文字，使用 `Text (verbatim)`。
- 文字数量越少越好。
- 按钮文案、数字、价格、进度值应清晰、普通、可替换。
- 固定艺术字标题可以更华丽，但必须明确是 fixed art title。
- 避免大量小字、长段文案、复杂说明。
- 避免中英文混杂过多。

适合生图的文字：

- 标题：`MERMAID PASS`
- 按钮：`CLAIM`
- 数字：`3200`
- 简短 label：`Level 12`

不适合生图的文字：

- 长规则说明
- 多行任务描述
- 很小的 UI 提示
- 未来要本地化的正文

## 5. 背景策略

背景应该建立主题气氛，但不能干扰 UI 拆解。

准则：

- 背景对比度低于前景 UI。
- 背景纹理不要穿透主面板。
- 背景装饰不要贴住按钮或卡片边缘。
- 可用柔和景深、暗化、低饱和方式降低背景干扰。

## 6. 风格一致性

prompt 中明确：

- 游戏品类
- 时代/题材
- 材质：金属、宝石、木头、玻璃、羊皮纸、科幻面板等
- 光照：柔光、霓虹、魔法光、自然光等
- 轮廓：厚描边、细描边、无描边

不要让同一界面混用太多风格，例如：

- 科幻 HUD + 中世纪羊皮纸按钮
- 写实背景 + 扁平卡通 UI
- 低饱和主面板 + 高饱和按钮但无视觉理由

## 7. Prompt 必备字段

建议每次包含：

- Use case：`ui-mockup`
- Asset type：游戏 UI 效果图
- UI 类型
- 画布比例和尺寸
- 主题/题材
- 组件清单
- 精确文字
- 可拆解约束
- 避免项

## 8. Negative / Avoid

常见避免项：

- 不要 device frame
- 不要 watermark
- 不要品牌 logo
- 不要真实人物
- 不要过多小字
- 不要乱码文字
- 不要把 UI 元素融合到背景里
- 不要强透视倾斜
- 不要让关键元素被裁切
- 不要过度模糊

## 9. 为后续 sprite 拆解优化

如果生成图明确要进入 `ui-sprite-regenerator`，要加上：

```text
所有 UI 元素应视觉上可分离，方便后续生成独立 sprite。
按钮底图、卡片底图、标题牌、icon、进度条、装饰件应有清晰边界。
动态文字应简短清晰，便于后续作为 Text Node 处理。
```

## 10. 质量判断

一张适合本项目的 UI 效果图，不只是漂亮，还应该满足：

- 可以指出 8-20 个明确 UI 元素
- 能区分哪些是背景、哪些是 UI、哪些是文字
- 大部分元素边界清楚
- 文字可读
- 后续可以写出 Sprite Plan
- 不需要大量人工猜测元素归属
